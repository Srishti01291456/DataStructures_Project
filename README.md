# Student Record Management System (DSA Project)

A Data Structures-based Student Record Management System designed to efficiently store, manage, and process student data using fundamental DSA concepts. The system enables fast operations on records while maintaining scalability and structured data handling.

# 📌 Project Overview

The system is designed in key functional stages:

## 🔹 1. Data Management
Stores student records dynamically

Supports structured storage using Linked Lists 

Maintains data such as name, roll number, marks, etc.

## 🔹 2. Data Processing & Operations
Performs CRUD operations (Create, Read, Update, Delete)

Enables searching and filtering of student records

Supports sorting based on marks or other attributes
## 💡 Key Features
 📂 Efficient data storage using Linked Lists 
 
 🔍 Fast search functionality for student records
 
 📊 Sorting using Merge Sort / Quick Sort
 
 👥 Supports multiple student records
## 🛠️ Tech Stack
Language: Java 

Concepts: Data Structures & Algorithms, OOP

Data Structures Used:
Linked List

Sorting Algorithms
## ▶️ How to Run
Step 1: Add Student Records
Enter student details via input
Data gets stored dynamically

Step 2: Perform Operations
Search student by roll number
Update or delete records
Sort students based on marks
## 📊 Working
Uses Linked List to store records dynamically

Implements search algorithms for quick retrieval

Uses Merge Sort / Quick Sort for ranking students
## 📈 Applications
🏫 Student Database Management

📊 Academic Record Systems

🧾 Result Processing Systems

🖥️ Basic ERP Systems

🤝 Contribution & Role

This project was developed as part of a Data Structures coursework. I contributed to:

Designing the complete system architecture

Implementing core data structures and algorithms

Developing CRUD operations and sorting logic

Optimizing performance and modularizing code
## ⚠️ Limitations
Limited scalability for very large datasets

No database integration (in-memory storage)

Basic UI (console-based system)
